## 1\. 버전(Version)이란?

소프트웨어 개발에서 **버전**은 프로그램이나 문서의 특정 시점을 구분하기 위한 번호 또는 이름을 의미합니다.  
예를 들어 우리가 자주 보는 **1.0 → 1.1 → 2.0** 같은 숫자 체계가 바로 버전입니다.

버전은 다음과 같은 상황에서 필요합니다.

-   프로그램의 **기능이 추가**되었을 때
-   **버그 수정**이나 **보안 패치**가 이루어졌을 때
-   기존 상태와 새로운 상태를 **구분**하고 싶을 때

즉, 버전은 프로젝트의 진화를 기록하는 이정표 역할을 합니다.

---

## 2\. 버전관리의 중요성

1.  **히스토리 추적**
    -   언제 어떤 변경이 있었는지 알 수 있어 유지보수가 쉬워집니다.
2.  **협업 효율성**
    -   여러 개발자가 동시에 작업할 때 충돌을 해결하고 코드를 병합할 수 있습니다.
3.  **백업 복구**
    -   문제가 있으면 바로 이전 버전으로 되돌릴 수 있습니다.

---

## 3\. 버전관리 시스템의 종류

버전관리 시스템(VCS: Version Control System)은 크게 **로컬, 중앙집중식, 분산식**으로 나눌 수 있습니다.

### (1) 로컬 버전관리(Local VCS)

-   개인 PC에서만 버전을 관리하는 방식
-   가장 단순하지만 협업에는 불편
-   예시: 파일 복사/백업, RCS(Revision Control System)

### (2) 중앙집중식 버전관리(Centralized VCS)

-   **중앙 서버**가 존재하고, 모든 사용자가 서버에서 파일을 받아 사용
-   협업이 가능하지만, 서버가 다운되면 작업에 큰 지장 발생
-   예시: CVS, Subversion(SVN)

### (3) 분산식 버전관리(Distributed VCS)

-   각각의 사용자가 **전체 저장소를 로컬에 복제**해 보관
-   네트워크가 끊겨도 로컬에서 작업 가능, 안정성 ↑
-   협업 및 브랜치 기능이 강력
-   예시: **Git, Mercurial**

---

## 4\. 가장 널리 쓰이는 버전관리 시스템 – Git

오늘날 가장 많이 사용되는 VCS는 단연 **Git**입니다.

-   오픈소스, 무료
-   분산식 구조 → 협업과 안정성 우수
-   GitHub, GitLab, Bitbucket 등 클라우드 기반 플랫폼과 결합하여 전 세계 개발자들이 사용
